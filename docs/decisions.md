# Decisions — databerganalytics

> Append-only. One line per decision. Date, decision, one-sentence reason. Read before making architectural choices.

- **2026-04-21** — Primary conversion = book-a-call; proposal-request and case-study download are secondary. Reason: a call is the honest first step for a consultancy; written proposals come after a call has happened.
- **2026-04-21** — Languages = EN + NL, English primary, Dutch as full parallel (not machine-translated). Reason: target market is NL-speaking but buyers often read in EN; half-translated sites read as cheap.
- **2026-04-21** — Tone = warm + quietly authoritative. Reason: differentiate from the dry "we deliver enterprise solutions" consultancy voice.
- **2026-04-21** — Radius philosophy = hard-edge (0) everywhere. Reason: aligns with Swiss precision; rounded corners would soften the whole brand.
- **2026-04-21** — Typography = Swiss precision pair (Neue Haas Grotesk / GT America for display + body, Berkeley Mono for numbers). Reason: matches the aesthetic direction and reads as competent without being loud.
- **2026-04-21** — Aesthetic direction = Swiss precision on an atmospheric dark canvas. Reason: user asked for both; the cohesive interpretation is Swiss layout/type with a dark palette, not a mashup of two visual languages.
- **2026-04-21** — Signature moment = interactive Optimization Dashboard ("dashboard professor"). Reason: dramatizes the core value proposition (finding and removing drag) in the hero — better than a static claim.
- **2026-04-21** — No chart library on first load; hand-rolled SVG in the dashboard. Reason: shipping a charting dep for one module bloats the bundle; revisit only if a second dynamic moment is added.
- **2026-04-21** — Roadmap kept to 15 phases with a strict "finish phase before advancing" gate. Reason: the failure mode for studio-grade sites is half-finished sections across many phases.
- **2026-04-21** — Wordmark = "databerganalytics" (lowercase, one word). Folder renamed from `databerganalaytics` (typo) to `databerganalytics`. Reason: user confirmed the one-word lowercase mark; fits Swiss precision aesthetic and keeps wordmark, domain, and folder identical.
- **2026-04-21** — Tailwind v4 CSS-first config over v3 + `tailwind.config.ts`. Tokens live in a `@theme` block inside `src/styles/tokens.css`; no JS Tailwind config file exists. CLAUDE.md §3 and §13 updated to reflect this. Reason: v4 is the current default, eliminates a config file, and keeps the single source of truth for tokens actually single.
- **2026-04-21** — Phase 1 verified on Node 25 via Homebrew (`brew install node pnpm`). `.nvmrc` still pins 22 (LTS). `pnpm install && pnpm check && pnpm build` all pass. Reason: user had Node 16 locally; 22-LTS is the commit target, 25 is fine to build on.

Will be resolved in Phase 7 when `src/content.config.ts` defines the schemas. Reason: adding the schema now without real content violates "don't build ahead of the phase."

Reason: hamburger on a 5-page site is overengineered; footer carries navigate for mobile; we can add a proper accessible menu in a later phase if real usage demands it.

CLAUDE.md §6 updated, commercial pair preserved in table as an optional future upgrade. Flagged `TODO(copy)` — user must confirm names/summaries before the service detail pages are written in Phase 6. - **2026-04-21** — `/nl/` home page deferred to Phase 12 despite the EN home being ready. Reason: the language toggle will 404 mid-build, which is acceptable — the project's phase-gated discipline requires Dutch copy to be human-written, not machine-translated or drafted-then-forgotten.
- **2026-04-21** — Phase 5 signature moment (Optimization Dashboard) ships **without absolute numeric values** (no €/hours/days/percentages). Bar widths represent qualitative "typical optimization shape"; delta labels are phrases ("cut meaningfully", "most of it gone", "near zero"). Reason: CLAUDE.md §8 forbids invented numbers, and we have no permissioned real-client outcomes yet. Data structure in `OptimizationDashboard.astro` is shaped so real values can be dropped in later without touching layout or animation.
- **2026-04-21** — Dashboard interactivity via inline `<script type="module">` in the component (no React/Svelte island). Reason: single small interactive surface; a framework island would be a dependency and a runtime cost for a ~60-line tab controller. Vanilla WAI-ARIA tablist with keyboard nav is the lightest shape.
- **2026-04-21** — Dashboard animation scope limited to bar-width CSS transitions. No counter tweening (no numbers to tween) and no chart-library dependency (hand-rolled rectangles). Reason: keeps the motion honest, avoids jank, respects `prefers-reduced-motion` cleanly, and matches CLAUDE.md §3 animation guidance.
- **2026-04-21** — `Hero.astro` (Phase 4's plain hero) retained but unused after Phase 5 swapped in the dashboard. Reason: may be reusable as a small hero on Services / About pages later; will be deleted in Phase 15 if still unused.
- **2026-04-21** — Service data lives in a typed TS module (`src/lib/services.ts`) rather than an Astro content collection. Reason: four services don't need markdown bodies or a CMS; a typed module is lighter and keeps the home grid, overview page, and detail template reading from one source. When the volume grows or fields get rich, revisit.
- **2026-04-21** — Service detail template structure: 01 Problem → 02 Approach → 03 Outcome → 04 Shape of engagement → Related work (placeholder) → Closing CTA → Other services. Reason: mirrors the actual decision path a buyer walks (why now / what do you do / what do I get / what does it cost in time/access / is there proof / how do I start), and gives the page a Swiss-precision editorial rhythm via numbered sections.
- **2026-04-21** — Engagement "What you keep" line on each service. Reason: differentiator for a consultancy — most competitors don't make it explicit that the client owns the output. Reinforces "no retainer, no lock-in" positioning stated in the outcome copy.

Three case studies written for Phase 7 drawing on those engagements, anonymized as "A Dutch health insurer" / "A retail banking group in EMEA" / "A global IT consulting partner." Reason: the CV provided 3 real engagements simultaneously; writing all three now removed a blocker for Phase 8's filter UX (which needs ≥2 items per filter axis to be meaningful). - **2026-04-21** — `src/content.config.ts` defines both `work` and `posts` collection schemas, even though `posts` is empty and deferred to Phase 11. Reason: one config file covers both, resolves the auto-generation deprecation warning comprehensively, and makes Phase 11 a copy-in of markdown files rather than a config change.

- **2026-04-21** — Astro template parser chokes on generic type arguments (`<'work'>`, `<typeof s>`) inside `{...}` expressions because it can't tell them apart from JSX. Resolution: move typed logic into the frontmatter, return plain objects to the template. `src/pages/work/index.astro` uses an `EnrichedCase` pre-computation for this reason. Lesson: keep generic-heavy TS out of Astro JSX-adjacent templates.

Reason: the placeholder was a known temporary state from Phase 6; this is the earliest phase with both pieces (schema + index) landed to wire it cleanly.

- **2026-04-21** — Contact-form architecture: hybrid Astro (static by default, one server route `/api/contact`). Form JS POSTs to that route; the route honeypots, validates, calls Turnstile siteverify, calls Resend `/emails`. Reason: matches CLAUDE.md §3 ("Astro API route → Resend for email. Honeypot field + Cloudflare Turnstile") exactly; Cloudflare end-to-end since Pages is a stated hosting target.
- **2026-04-21** — Manual validation in the endpoint, no Zod dependency. Reason: 4 fields (name/email/organization/message) + honeypot + Turnstile token don't justify a validation library. `EMAIL_RE` + length checks cover it.
- **2026-04-21** — Form gracefully degrades when env vars are missing. No `RESEND_API_KEY` → endpoint returns 503 with helpful error. No `PUBLIC_TURNSTILE_SITE_KEY` → client skips widget, endpoint skips siteverify. Reason: dev workflow should not require six external services configured before the contact page is usable; production failure modes should be informative.
- **2026-04-21** — Honeypot field named `website` in a visually-hidden wrapper (`absolute -left-[10000px]`). On bot-fill, endpoint returns OK without sending mail. Reason: typed-bot traffic targets every input; pretending success hides the trap from adversarial feedback loops.
- **2026-04-21** — `BaseLayout.astro` gained `<slot name="head" />` in Phase 10 so the contact page can inject the Turnstile script only where needed. Reason: loading a Cloudflare Challenges script on every page would waste a network request on every non-contact URL; scoping it is correct.

Reason: Mono renders above-the-fold on nearly every page (numbered section tags, KPI labels on the dashboard, monospace eyebrows); without preload it was downloaded via `@font-face` discovery after first paint and briefly swapped. 49 KB crossorigin preload is a fair trade for eliminating the FOUT on mono glyphs.

All schemas linked via `@id` so crawlers can resolve publisher/provider nodes. Reason: Google values the Organization node being reachable from every page, but doesn't want duplicate type declarations; scoping by page type keeps payload tight.
- **2026-04-21** — Phase 14 OG images = Satori pipeline. `scripts/build-og.mjs` runs before `astro build` and emits 33 PNGs (one per route, EN + NL). Design: dark canvas, Geist wordmark, mono kicker, accent rule — Swiss-precision, matches the site. Dependencies added: `satori` + `@resvg/resvg-js` (both devDeps, build-time only). Fonts: Geist OTF + GeistMono OTF committed to `scripts/og/fonts/` (SIL OFL; OTF required because Satori does not decompress woff2). Reason: Satori scales cleanly as new insights/cases land and uses the real Geist typeface (matches the site exactly); hand-rolled SVG substitution was the alternative, rejected because it requires manual re-run per content add. Generated PNGs gitignored — Cloudflare regenerates on every build.
- **2026-04-21** — Phase 14 sitemap filters `/dev/ui` (visual-QA page). `robots.txt` disallows `/dev/` and `/api/`. Reason: the QA page is noindex already but should not even appear in the sitemap; `/api/contact` is a form endpoint, not a page, and should not be crawled.
